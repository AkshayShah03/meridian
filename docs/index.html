The Problem

AI assistants are fundamentally broken

Most AI products fail in the same predictable ways. Meridian is architected to solve each one.

🌀
Hallucination
Models fabricate facts when they don't know the answer. No ground truth enforcement.
🕰️
Stale Knowledge
Training cutoffs mean models can't answer "What's happening now?" accurately.
🧠
No Long-Term Memory
Every session starts blank. Users repeat themselves. Context is lost.
🔮
Zero Observability
When something breaks — which model, which token? A black box with no trace.
🧱
Tight Coupling
Swapping a model or adding a tool requires rewriting the entire backend.
The Solution

Meridian's five-layer answer

Each design decision maps directly to one of the problems above.

01 · Anti-Hallucination
Tool-First Routing
Factual queries never reach the LLM alone. The ReAct agent loop calls authoritative tools first: Google News RSS for headlines, IANA tzdata for time, AST-based calculator for arithmetic. Anything that can be verified against a live source is verified. The model only synthesises.
ReAct LoopTool RuntimeAST Calculator
02 · Real-Time Data
Live Source Integration
The tool runtime fetches live data on every request. News from Google News RSS at query time. Timezone data from the IANA tzdata package, not model weights. Date and time always accurate.
Google News RSSIANA tzdataDuckDuckGo
03 · Semantic Memory
Multi-Layer Memory Architecture
Redis short-term history, pgvector semantic recall across all past conversations, episodic summaries to compress long histories. Fully decoupled from inference — runs over Kafka.
pgvector 768-dimnomic-embed-textRedis
04 · Full Observability
Five-Layer Tracing Stack
Every request generates OpenTelemetry traces into Jaeger, metrics into Prometheus/Grafana, LLM traces into Langfuse. An independent eval-service scores every response across 5 dimensions and writes to Postgres.
OpenTelemetryJaegerLangfuseGrafana
05 · Zero Coupling
Kafka-Native Event Bus
No service calls another service directly. Every interaction flows through named Kafka topics. Swap Ollama for GPT-4 or replace the eval algorithm — zero other services change.
Apache Kafka8 Topicsaiokafka
Bonus · Efficiency
Prompt Caching + Multi-Model Routing
Inference gateway caches prompt prefixes in Redis by SHA-256 hash. Cache hits skip inference entirely, cutting TTFT to single-digit milliseconds. Routing selects llama3.2:3b for simple, mistral:7b for complex, Anthropic Claude as fallback.
Redis Cachellama3.2:3bmistral:7bClaude

Services

8 independent microservices

Each service owns exactly one responsibility and communicates only through Kafka.

auth-service
JWT registration, login, token refresh, validation. Stateless — verifiable by any service without a network call.
FastAPIasyncpgbcryptPyJWT
ws-gateway
WebSocket endpoint. Publishes user messages to Kafka and fans streamed response chunks back to the right session.
FastAPIaiokafkaWebSockets
memory-service
Redis short-term history, pgvector semantic recall, episodic summaries with tiktoken-aware chunking.
aiokafkaasyncpgpgvectortiktoken
inference-gateway
Multi-model routing, prompt caching, Ollama token streaming, structured output via Instructor, Anthropic fallback.
aiokafkahttpxinstructoranthropic
agent-orchestrator
ReAct reasoning loop. Keyword pre-router selects tool vs. direct inference. Multi-step chains with Redis state.
aiokafkahttpxRedis state
tool-runtime
AST calculator (no eval()), DuckDuckGo + Google News search, SSRF-protected URL fetch, IANA timezone datetime.
aiokafkahttpxtzdataAST
eval-service
Automatic quality scoring on every response: coherence, relevance, completeness, latency, safety. Written to Postgres.
aiokafkaasyncpgPostgres
notify-service
Notification dispatch consumer. Subscribes to audit and eval topics to trigger alerts without coupling to core flow.
FastAPIaiokafka
