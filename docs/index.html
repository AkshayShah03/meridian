The Problem

AI assistants are fundamentally broken

Most AI products fail in the same predictable ways. Meridian is architected to solve each one.

🌀
Hallucination
Models fabricate facts, numbers, and events when they don't know the answer. No ground truth enforcement.
🕰️
Stale Knowledge
Training cutoffs mean models can't answer "What's happening now?" or compute today's date accurately.
🧠
No Long-Term Memory
Every session starts blank. Users repeat themselves. Context is lost across conversations.
🔮
Zero Observability
When something breaks — which model, which token, which service? A black box with no trace.
🧱
Tight Coupling
Swapping a model or adding a tool requires rewriting the entire backend.
The Solution

Meridian's five-layer answer

Each design decision maps directly to one of the problems above.

01 · Anti-Hallucination
Tool-First Routing
Factual queries never reach the LLM alone. The ReAct agent loop calls authoritative tools first: Google News RSS for headlines, IANA tzdata for time, AST-based calculator for arithmetic. Anything that can be verified against a live source is verified. The model only synthesises.
ReAct LoopTool RuntimeAST Calculator
02 · Real-Time Data
Live Source Integration
The tool runtime fetches live data on every request. News is pulled from Google News RSS at query time. Timezone data comes from the IANA tzdata package, not a model weight. Date and time are always accurate.
Google News RSSIANA tzdataDuckDuckGo
03 · Semantic Memory
Multi-Layer Memory Architecture
Redis short-term history for the current session, pgvector semantic similarity recall across all past conversations, and episodic summaries to compress long histories. Fully decoupled from inference — runs over Kafka.
pgvector 768-dimnomic-embed-textRedis
04 · Full Observability
Five-Layer Tracing Stack
Every request generates traces through OpenTelemetry into Jaeger. Metrics flow to Prometheus and Grafana. LLM-specific traces go to Langfuse. An independent eval-service automatically scores every response across 5 dimensions, written to Postgres.
OpenTelemetryJaegerLangfuseGrafana
05 · Zero Coupling
Kafka-Native Event Bus
No service calls another service directly. Every interaction flows through named Kafka topics. Swap Ollama for GPT-4 or replace the eval algorithm: zero other services change.
Apache Kafka8 Topicsaiokafka
Bonus · Efficiency
Prompt Caching + Multi-Model Routing
The inference gateway caches prompt prefixes in Redis using SHA-256 hashes. Identical prompts skip re-tokenisation, cutting TTFT significantly. Routing selects llama3.2:3b for simple queries, mistral:7b for complex ones, Anthropic Claude as premium fallback.
Redis Cachellama3.2:3bmistral:7bClaude

Request Flow

A message, step by step

"What time is it in Tokyo right now?" — with agent mode on.

1
Browser → WS Gateway
Message arrives over WebSocket. JWT validated inline — stateless HS256, no auth service call needed.
ws://gateway:8002
2
Gateway publishes to user-events
Event contains message and session ID. No downstream service is called directly.
3
Memory Service recalls context
Retrieves recent Redis history and runs a pgvector cosine similarity search (nomic-embed-text, 768-dim) to surface relevant past conversations.
cosine similarity · pgvector IVFFlat
4
Agent Orchestrator runs ReAct loop
Detects "time" via keyword pre-router, dispatches to tool-requests. Intermediate state stored in Redis between iterations.
Reason → Act → Observe → Reason…
5
Tool Runtime executes datetime lookup
Reads from IANA tzdata — bundled, always accurate. Returns exact local time. No LLM involved, zero hallucination risk.
tzdata · IANA database · Python zoneinfo
6
Inference Gateway synthesises and streams
Tool result injected into prompt. Redis prompt cache checked first (SHA-256). On miss: streams token-by-token from Ollama, each chunk published to llm-responses.
Cache hit → 0ms inference · Miss → llama3.2:3b stream
7
WS Gateway fans chunks to browser
Consumes response stream from Kafka, pushes each token over WebSocket to exactly the session that originated the message.
8
Eval Service scores the response
After streaming completes: coherence, relevance, completeness, latency, safety. Written to Postgres for trend analysis.
5 dimensions · Postgres audit log
